# CMSC313-Project-3
O0 vs. O1 vs. O3:

O0 has no optimization which can be seen through the length of the file being much bigger than the other two at 253 lines. It is also a lot more human readable than the other optimizations. It is also theoretically slower than the rest but with how simple the program is its hard to tell.

O1 has some optimization mostly favoring compile time which can be seen through the length of the file being the smallest with the fewest lines of the three files at 157 lines. It is also harder to tell what the assembly is actually doing. It is also supposed to compile and run faster than O0 but with how simple the program is its hard to tell.

O3 has more optimization with a focus on speed which can be seen through the length of the file being smaller than that of the O0 file. Interestingly enough it is longer than the O1 file at 173 lines. This is from it attempting to make the file even faster but again with how simple the program is there is no noticeable difference in speed. Additionally once again it is harder to read what it is actually doing.

Overall I would say for this project O1 is probably the best of the bunch as it has the fewest lines and is relatively the same speed as the others.
